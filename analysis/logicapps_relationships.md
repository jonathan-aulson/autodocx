### Observed Gaps

1. **Missing relationship graph:** Steps reference HTTP URLs, Dataverse tables, SQL connectors, and child flows, but none are transformed into explicit relationship objects. Graph builder and LLM prompts therefore lack call/dependency edges.
2. **Inconsistent connector metadata:** The only indicator of dependency type is the free-form `connector` string (e.g., `shared_sql`, `shared_commondataservice`). There is no normalized `target_kind` or CRUD verb.
3. **No evidence linkage per dependency:** Evidence anchors apply to the whole workflow; we need per-relationship `evidence` so the renderer and LLM can cite specific step lines.

## Shared Relationship Schema (Draft)

To keep extractors, the Option1 mapper, renderers, and LLM context builder aligned, every signal that can describe dependencies (Logic Apps workflows, Azure Functions, pipelines, SQL objects, etc.) should attach a `relationships` array with the following shape:

```json
{
  "relationships": [
    {
      "id": "step-unique-id",
      "source": {
        "type": "trigger|action|job|function",
        "name": "Compose_Empty_Line_Items",
        "step_id": "Compose_Empty_Line_Items",
        "component": "TownePark.Billing"
      },
      "target": {
        "kind": "http|dataverse|sql|storage|queue|workflow|service",
        "ref": "shared_sql:BellServiceFeeAmount",
        "display": "SQL (shared_sql) - BellServiceFeeAmount stored proc",
        "component": "Billing DB"
      },
      "operation": {
        "type": "calls|reads|writes|invokes|publishes|subscribes",
        "verb": "POST",
        "crud": "read|create|update|delete|execute",
        "protocol": "https|sql|dataverse|servicebus"
      },
      "connector": "shared_sql",
      "direction": "outbound|inbound",
      "auth": {
        "connection_reference": "shared_sql",
        "auth_level": "function|anonymous|managed_identity"
      },
      "context": {
        "url_or_resource": "https://api.businesscentral.dynamics.com/.../datasets/default",
        "child_workflow": "BillableAccountsChildFlow",
        "table": "msdyn_customerinvoices",
        "environment": "prod",
        "notes": "Auto-generated by LogicAppsWDLExtractor"
      },
      "roles": ["data.reads", "workflow.invokes"],
      "evidence": [
        "out/sir/BellServiceFeeChildFlow-...json.json:step:Get_BellServiceFeeAmount"
      ],
      "confidence": 0.95
    }
  ]
}
```

### Field Guidance

| Field | Required? | Description | Producing component(s) |
| --- | --- | --- | --- |
| `id` | ✅ | Stable identifier (e.g., `${step_id}:${target_kind}:${hash(target_ref)}`) so downstream joins are deterministic. | Extractors |
| `source.*` | ✅ | Metadata about the originating trigger/action/function; include `component` once mapper resolves it. | Extractors (step info), Mapper (component enrichment) |
| `target.kind` | ✅ | Enum describing dependency class (`http`, `sql`, `dataverse`, `sharepoint`, `workflow`, `function`, `queue`, `event`, `pipeline`, `table`). | Extractors normalize |
| `target.ref` | ✅ | Canonical identifier (URL, table name, workflow ID). Enables joiners and renderer labels. | Extractors |
| `operation.type` | ✅ | Relationship semantics consumed by graph builder + renderer (e.g., `calls`, `reads`, `writes`, `invokes`). | Extractors map connectors to verbs |
| `operation.verb` | optional | HTTP verb, SQL action, CRUD label (helps LLM prompts). | Extractors |
| `connector` | ✅ | Original Logic Apps connector name for traceability. | Extractors |
| `direction` | ✅ | `inbound` or `outbound` from the workflow perspective; simplifies narrative phrasing. | Extractors |
| `auth.*` | optional | Captures connection references or auth modes for security/UX copy. | Extractors |
| `context.*` | optional | Connector-specific metadata (URL, child workflow name, Dataverse table, parameter hints). | Extractors |
| `roles` | optional | Precomputed role tags feeding mapper + LLM context (e.g., `data.reads`). | Extractors / role inference |
| `evidence` | ✅ | File/line anchors or `step:<name>` markers so renderer + LLM can cite. | Extractors |
| `confidence` | optional | Numeric facet used by mapper to prioritize narratives. | Extractors |

### Downstream Consumption

- **Mapper (`artifacts.option1`):** Copy `relationships` into each workflow artifact and build a `relationship_matrix` summarizing counts by `operation.type` + `target.kind`. Surface inbound/outbound UX flows using `source.name -> target.display`.
- **Renderer (`business_renderer`, `mkdocs`):** Render UX narratives (trigger -> action -> dependency), dependency tables grouped by `target.kind`, and highlight data touchpoints (e.g., “Writes to Dataverse table X via shared_commondataservice”). Use `evidence` to link callouts back to steps.
- **LLM Context Builder (`llm/context_builder.py`):** Include trimmed `relationships` (or `relationship_matrix`) in component contexts so prompts can ask for “UX story + dependencies”. Provide aggregated counts and notable targets (top HTTP endpoints, Dataverse tables, SQL objects).
- **Graph Builder / Joiners:** Use `target.kind` + `target.ref` to instantiate edges between workflow nodes and existing API/DB nodes, improving diagrams and impact analysis.
